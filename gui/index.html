Setup Guide

Follow these steps to configure your relay proxy

1

Choose Your Auth Key

Pick a secret password. This same key must be used in both the app and your Apps Script.

2

Create Google Apps Script

  1. Go to script.google.com and click New Project
  2. Delete all the default code in the editor
  3. Paste the code you copied in Step 1 Ctrl+V
  4. Click Deploy → New deployment
  5. Click the ⚙️ gear icon → select Web app
  6. Set Execute as: → Me
  7. Set Who has access: → Anyone
  8. Click Deploy and copy the Deployment ID
For load balancing, repeat this process multiple times with different Google accounts. All deployments must use the same Auth Key.
3

Add Deployment ID

Paste the Deployment ID you copied from Apps Script:

No deployments added yet

4

Connect!

Go to the Home tab and click Connect. Your proxy will be ready at:

HTTP → 127.0.0.1:8085 SOCKS5 → 127.0.0.1:1080

Configure your browser or system proxy settings to use these addresses.

Installation Guide

  1. Click Download Certificate above to save ca.crt
  2. Double-click the downloaded ca.crt file
  3. Click Install Certificate
  4. Select Local Machine (or Current User)
  5. Choose Place all certificates in the following store
  6. Click Browse and select Trusted Root Certification Authorities
  7. Click OK, then Next, then Finish
After installation, restart your browser for the changes to take effect.
  1. Click Download Certificate above to save ca.crt
  2. Double-click the downloaded file — Keychain Access will open
  3. The certificate will appear in System keychain
  4. Double-click the certificate MasterHttpRelayVPN
  5. Expand Trust section
  6. Set When using this certificate to Always Trust
  7. Close the window and enter your password to confirm
Or use terminal: sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain ca.crt
  1. Click Download Certificate above to save ca.crt
  2. Copy to system CA store: sudo cp ca.crt /usr/local/share/ca-certificates/mastervpn-ca.crt
  3. Update CA store: sudo update-ca-certificates
On Fedora/RedHat, use /etc/pki/ca-trust/source/anchors/ and run sudo update-ca-trust
  1. Click Download Certificate above to save ca.crt
  2. Open Firefox — go to Settings (about:preferences)
  3. Search for Certificates in the search bar
  4. Click View Certificates
  5. Go to the Authorities tab
  6. Click Import and select the downloaded ca.crt
  7. Check Trust this CA to identify websites

Firefox uses its own certificate store — it ignores system certificates by default.
  1. Chrome and Edge use the system certificate store
  2. Follow the Windows or macOS guide above to install the certificate system-wide
  3. After installation, restart Chrome / Edge
  4. Navigate to chrome://settings/certificates to verify
If you still see warnings, try clearing the browser cache and DNS: chrome://net-internals/#dns

Why is this needed?

In Apps Script mode, the proxy decrypts and re-encrypts HTTPS traffic locally. A trusted CA certificate prevents security warnings in your browser.

  • The certificate is local only — it never leaves your machine
  • A new certificate is generated automatically on first run
  • You can Regenerate it anytime from this page
  • Each certificate is valid for 10 years
